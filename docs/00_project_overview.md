# 00. プロジェクト概要

## 1. プロジェクト名

Aurens HR Platform

## 2. 目的

既存勤怠管理システムと連携し、人事労務業務をWeb上で一元管理するシステムを構築する。

本システムは、従業員マスターを既存勤怠管理システム側に持たせ、人事労務側では追加情報、申請、入社手続き、給与明細、マイナンバー状況、e-Gov電子申請を管理する。

## 3. 背景

現在、勤怠管理システムには従業員マスターが存在している。
そのため、新しい人事労務システム側で従業員マスターを重複管理すると、氏名、所属、雇用区分、勤務パターン、退職情報などの不整合が発生する可能性がある。

この問題を避けるため、既存勤怠管理システムを正マスターとし、職員IDをキーにして人事労務システムと連携する。

## 4. システム化対象

- 人事労務
- 入社手続き
- 従業員情報の申請
- Web給与明細
- マイナンバー管理
- e-Gov電子申請
- 権限管理
- 通知管理
- ファイル管理
- 操作ログ管理
- CSV入出力

## 5. 開発方針

本システムはWebシステムとして開発する。
PCおよびスマートフォンから利用できる構成とする。

初期開発では、人事労務、入社手続き、従業員申請、Web給与明細を中心に構築し、e-Gov電子申請は最終的に送信まで対応する。

マイナンバーは初期版では番号そのものを保存せず、提出状況のみ管理する。ただし、将来的に暗号化保存できるように設計する。

## 6. 設計思想

### 6.1 勤怠マスターを正マスターとする

従業員情報は既存勤怠管理システムを基準とする。
人事労務システム側では、勤怠側に存在しない追加情報のみ保持する。

### 6.2 設定で動くシステムにする

申請フォーム、承認ルート、通知、権限、帳票、カスタム項目は、できる限り管理画面から設定できるようにする。

### 6.3 将来拡張を前提にする

給与計算、年末調整、人事評価、健康診断、ストレスチェック、AI支援機能などを将来的に追加できる構成とする。

### 6.4 監査性を確保する

誰が、いつ、何を、どのように変更したかを記録する。
申請、承認、差戻し、閲覧、CSV出力、ファイルダウンロードなどもログ対象とする。
