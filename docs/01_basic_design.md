# 01. 基本設計書

## 1. システム概要

Aurens HR Platform は、既存勤怠管理システムと連携するWeb型人事労務システムである。

従業員マスターは既存勤怠管理システムを正マスターとし、人事労務システム側では、従業員に紐づく追加情報、入社手続き、従業員申請、Web給与明細、マイナンバー提出状況、e-Gov電子申請を管理する。

## 2. 利用者

- システム管理者
- 人事担当者
- 給与担当者
- 部門管理者
- 一般従業員
- 入社予定者

## 3. 基本機能

### 3.1 勤怠管理システム連携

既存勤怠管理システムと連携し、以下の情報を取得する。

- 職員ID
- 氏名
- 所属部署
- 雇用区分
- 勤務パターン
- 役職
- 入社日
- 退職日
- 在職区分
- メールアドレス

職員IDを連携キーとする。

### 3.2 人事労務管理

勤怠側にない人事労務情報を管理する。

- 住所
- 連絡先
- 扶養家族
- 社会保険
- 雇用保険
- 契約情報
- 資格
- 免許
- 通勤情報
- 添付ファイル
- 自由項目

### 3.3 入社手続き

入社予定者にWebフォームを送付し、本人が必要情報を入力する。

主な機能は以下とする。

- 入社予定者登録
- 招待メール送信
- 本人入力フォーム
- 一時保存
- 必須書類アップロード
- 人事確認
- 差戻し
- 承認
- 従業員情報への反映
- e-Gov電子申請への連携

### 3.4 従業員申請

従業員本人が自身の情報変更を申請できる。

対象例は以下とする。

- 住所変更
- 氏名変更
- 扶養追加
- 扶養削除
- 銀行口座変更
- 緊急連絡先変更
- 資格登録
- 免許更新
- その他申請

申請フォームは管理者が設定できる。

### 3.5 Web給与明細

給与計算は当面外部システムで行う。
本システムでは、外部給与システムから出力したCSVを取り込み、Web給与明細を発行する。

主な機能は以下とする。

- 給与CSV取込
- 取込チェック
- 明細レイアウト設定
- 公開日時設定
- 従業員閲覧
- PDF表示
- 過去明細閲覧
- 公開通知

### 3.6 マイナンバー管理

初期版ではマイナンバー番号そのものは保存しない。
提出依頼、提出済、確認済、不備、不要などのステータスのみ管理する。

将来版では、暗号化保存、閲覧権限、操作ログ、自動削除、二段階認証に対応できるように設計する。

### 3.7 e-Gov電子申請

最終的にe-Gov電子申請の送信まで対応する。

段階的に以下の開発を行う。

#### 第1段階

- 届出データ作成
- 入力チェック
- 電子申請一覧管理

#### 第2段階

- e-Gov API連携
- 送信
- 送信結果取得

#### 第3段階

- 公文書取得
- エラー再申請
- 申請履歴管理

## 4. 権限管理

権限はユーザーごと、またはロールごとに設定できる。
固定権限ではなく、機能単位で細かく付与できる構成とする。

主な権限項目は以下とする。

- 従業員閲覧
- 従業員編集
- 申請承認
- 給与明細管理
- マイナンバー状況閲覧
- e-Gov送信
- CSVインポート
- CSVエクスポート
- システム設定
- ファイル閲覧
- ファイル削除

## 5. 通知機能

以下の通知を行う。

- 入社手続き依頼
- 入社手続き提出
- 申請提出
- 承認
- 差戻し
- 給与明細公開
- 契約期限
- 資格期限
- 在留期限
- e-Govステータス変更

## 6. 操作ログ

以下の操作を記録する。

- ログイン
- ログアウト
- 閲覧
- 登録
- 更新
- 削除
- 承認
- 差戻し
- CSV出力
- ファイルダウンロード
- e-Gov送信
