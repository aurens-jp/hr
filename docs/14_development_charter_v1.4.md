# Aurens HR Platform 開発憲章 Ver1.0

## 1. 私たちが作るもの

Aurens HR Platform は、単なる人事労務システムではない。  
人事、勤怠、給与、電子申請、申請承認、AI支援を統合し、会社の人に関する業務を一元化するクラウドHRプラットフォームである。

## 2. 開発の目的

現場担当者の作業負担を減らし、情報の二重管理をなくし、従業員本人・管理者・人事担当者が迷わず使える環境を提供する。

## 3. 大切にする価値

### 3.1 迷わない

画面を見れば次に何をすればよいか分かることを重視する。

### 3.2 二重管理しない

既存勤怠システムを正マスターとし、同じ情報を複数システムで別々に管理しない。

### 3.3 設定で変えられる

申請フォーム、承認ルート、通知、権限、帳票は設定で変更できることを目指す。

### 3.4 記録が残る

誰が、いつ、何を変更したかを必ず残す。

### 3.5 AIと一緒に育てる

設計書、画面ID、API ID、テーブル定義を明確にし、AIが迷わず実装できる状態を保つ。

## 4. 開発で優先すること

1. 正確性
2. セキュリティ
3. 業務に合うこと
4. 使いやすさ
5. 拡張性
6. 開発速度

## 5. 初期段階でやらないこと

- すべての機能を一度に作る
- 勤怠マスターを人事側で重複管理する
- マイナンバー番号そのものを初期版で保存する
- 権限を大雑把に固定する
- 設計書なしに実装する
- GitHubに秘密情報を保存する

## 6. 判断に迷ったときの基準

- 現場担当者の手間は減るか
- 将来の運用変更に耐えられるか
- 監査・履歴が残るか
- AIや開発者が理解しやすいか
- 個人情報を安全に扱えるか
- 既存勤怠システムとの整合性が保てるか

## 7. 設計書を資産とする

コードだけでなく、設計書も重要な資産とする。  
GitHub上のドキュメント、画面設計、DB設計、API設計、Claude Code用ルールは、常に実装と一致するよう更新する。
