# Git Rules

## Branches

- main
- develop
- feature/*
- fix/*
- hotfix/*
- release/*

## Commit Messages

Use Conventional Commits.

Examples:

```text
feat: add employee list screen
fix: correct permission check
docs: add development charter
db: add employee request tables
api: add employee search route
```

## Pull Request

Each PR should include:

- purpose
- changed files
- test result
- related issue
- notes
