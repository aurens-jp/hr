# 00_プロジェクト概要

## 1. プロジェクト名

Aurens HR Platform

## 2. プロジェクト目的

Aurens HR Platform は、既存勤怠管理システムと連携するWeb人事労務システムである。

従業員マスターは既存勤怠管理システムを正マスターとし、人事労務システム側では、勤怠管理に存在しない追加情報、入社手続き、従業員申請、給与明細、マイナンバー提出状況、e-Gov電子申請等を管理する。

## 3. 背景

現在、人事労務関連業務は以下のように複数業務に分かれている。

- 従業員情報管理
- 入社時の書類収集
- 住所変更、扶養変更、口座変更等の申請
- 給与明細配布
- マイナンバー提出状況管理
- 社会保険、雇用保険の届出
- e-Gov電子申請
- 各種期限管理
- 帳票作成

これらをWebシステム上で一元管理し、業務効率化、入力ミス削減、履歴管理、権限管理、電子申請対応を実現する。

## 4. システムの基本思想

### 4.1 勤怠マスターを正マスターとする

従業員の基本情報は既存勤怠管理システムを正とする。

本システムでは職員IDをキーとして勤怠側の従業員情報を参照または同期し、人事労務に必要な追加情報のみ保持する。

### 4.2 設定で運用できるシステムにする

申請フォーム、承認ルート、通知、帳票、権限、カスタム項目は、プログラム改修ではなく管理画面上の設定で変更できる設計とする。

### 4.3 将来拡張を前提にする

初期版では人事労務を中心に構築するが、将来的に以下の機能を追加できるようにする。

- 給与計算
- 年末調整
- 人事評価
- 健康診断管理
- ストレスチェック
- 組織図
- 電子契約
- AI書類チェック
- AIチャット
- AI分析

### 4.4 監査性を重視する

人事労務情報は重要情報であるため、操作ログ、承認履歴、変更履歴、閲覧履歴を保持する。

特に、マイナンバー、給与明細、社会保険情報、扶養情報、電子申請情報については厳格な権限制御を行う。

| 利用者 | 主な役割 |
|---|---|
| システム管理者 | 全体設定、権限管理、マスター管理 |
| 人事担当者 | 従業員情報、入社手続き、申請承認、電子申請 |
| 給与担当者 | Web給与明細、給与CSV取込 |
| 部門管理者 | 自部門の従業員情報確認、申請承認 |
| 従業員 | 自分の情報確認、申請、給与明細確認、入社手続き |
